# Flip 54

A workout app that turns a standard 54-card deck into a randomized fitness session. Flip a card, do the reps, flip again. The deck is the workout.

## How it works

Each card prescribes an exercise based on its **suit** (body region) and **rank** (rep count or hold).

| Suit | Body focus |
|------|------------|
| ♥ Hearts   | Lower body  |
| ♠ Spades   | Upper body  |
| ♣ Clubs    | Total body  |
| ♦ Diamonds | Core        |
| 🃏 Joker   | Jumping jacks |

- **Numbered cards (2–10):** pip value × difficulty multiplier.
- **Face cards (J, Q, K):** 10 reps × difficulty.
- **Aces:** 60-second hold (push-up hold, dead hang, wall sit, plank, hollow body — depends on suit and equipment).
- **Joker:** 40 jumping jacks.

Available equipment expands the exercise pool:

- **Body-weight only** is always supported.
- **Weights** mix in goblet squats, biceps curls, shoulder press, triceps extensions, weighted sit-ups, and thrusters.
- **Pull-up bar** adds pull-ups and dead hangs.

Pick **Beginner / Standard / Advanced** difficulty to scale reps and hold seconds. A **Half Deck** option (27 cards: face cards, high pips, and one joker) shortens the session without softening it.

## Project layout

```
Flip54.xcodeproj         Xcode project (generated by XcodeGen from project.yml)
Flip54/                  iOS app source
  App/                   App entry, root ContentView
  Components/            Reusable card rendering
  Design/                DS color/font/layout tokens
  Features/              Per-screen views (PreWorkout, ActiveWorkout, Settings, History, Profile, Completion, Onboarding)
  Resources/             Bundled fonts and deck artwork directories
  Utilities/             Haptics, audio, accessibility helpers
Packages/
  Flip54Core/            Pure domain types — Card, Suit, Rank, Difficulty, Equipment, Exercise, Prescription
  Flip54Storage/         SwiftData models — UserSettings, WorkoutHistory, ActiveSession + ActiveSessionStore
  Flip54WorkoutEngine/   WorkoutCoordinator state machine, HoldTimer
project.yml              XcodeGen project spec
```

## Requirements

- Xcode 16 or later
- iOS 17.0+ deployment target
- Swift 6.0 with `SWIFT_STRICT_CONCURRENCY: complete`
- [XcodeGen](https://github.com/yonaskolb/XcodeGen) (only when adding source files)

## Build & run

The Xcode project is checked in, so cloning and opening `Flip54.xcodeproj` is enough for most contributors.

```sh
git clone git@github.com:zwcampbell/flip-54-app.git
cd flip-54-app
open Flip54.xcodeproj
```

Build for the simulator from the command line:

```sh
xcodebuild -scheme Flip54 \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' \
  build
```

If you add new source files, regenerate the project before building:

```sh
xcodegen generate
```

## Tests

The pure-domain packages have unit tests under `Packages/*/Tests`. Run them via SwiftPM:

```sh
cd Packages/Flip54Core && swift test
cd Packages/Flip54Storage && swift test
cd Packages/Flip54WorkoutEngine && swift test
```

`Flip54Core` carries an exhaustive 33-case prescription matrix that covers every (suit × rank × equipment × difficulty) combination plus joker handling and ace holds.
